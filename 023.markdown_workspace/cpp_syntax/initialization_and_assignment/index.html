初始化和赋值(initialization_and_assignment)

todo: 当前是一个完整的初始化和赋值文档，进一步精简一个只涉及类类型的(面向对象的类设计具体的实践)

1. 初始化(Initialization)

  • 变量的初始化在构造时提供其初值。
  • 初值可以由声明符 或 new 表达式的初始化器部分提供。函数调用也会发生：形参、返回值 也会被初始化。
  • 对于每个声明符，初始化器必须是下列之一：
    (1) = 表达式
    (2) = {} ; = { 初始化列表 } ； = { 指派初始化器列表 }
    (3) ( 表达式列表 ) ; ( 初始化器列表 )
    (4) {} ; { 初始化器列表 } ; { 指派初始化器列表 }
    1） 复制初始化语法。
    2） 列表初始化语法(c11)。聚合初始化语法(c11前)。
    3） 直接初始化语法。
    4） 列表初始化语法。
    表达式[1]：（1.不带括号的，2.逗号表达式除外的）任意表达式。
    表达式列表：表达式[1] 的逗号分隔列表。
    初始化器列表：初始化器子句[2] 的逗号分隔列表。
    指派初始化器列表：指派初始化器子句[2] 的逗号分隔列表。

  • 初始化器子句[2]
    (1) 表达式
    (2) {}
    (3) { 初始化器列表 }
    (4) { 指派初始化器列表 }
    语法(2)(4)统称花括号包围的初始化器列表。

  • 初始化语义
    如果没有为对象指定初始化器，对象会 默认初始化。如果没有为对象指定初始化器，程序非良构。
    如果为对象指定的初始化器是 ()（由于语法限制不能在声明符中出现），那么该对象会值初始化。如果为引用指定的初始化器是 ()，那么程序非良构。
    初始化器语义定义：

  • 初始化引用，另见。
  • 初始化对象。类型T:
    • 使用语法(1)的初始化器，对象会 复制初始化。
    • 使用语法(2)(4)的初始化器，对象会 列表初始化。
    • 使用语法(3)的初始化器，对象会 直接初始化。
#include <string>

std::string s1;           // 默认初始化
std::string s2();         // 不是初始化！
                          // 实际上声明了没有形参并且返回 std::string 的函数 “s2”
std::string s3 = "hello"; // 复制初始化
std::string s4("hello");  // 直接初始化
std::string s5{'a'};      // 列表初始化（C++11 起）

char a[3] = {'a', 'b'}; // 聚合初始化（C++11 起是列表初始化的一部分）
char& c = a[0];         // 引用初始化

1.1 直接初始化(Direct initialization)

以一组明确的构造函数实参对对象进行初始化。 * 语法
1）T obj(实参);
2) T obj{实参}；
3) T(其他对象)；T(实参1,实参2...);
4) static_cast(其他对象)；
5) new T(实参列表)
6) 类::类：成员(实参列表){};
7) 实参{};
(1) 表达式或花括号初始化列表 的非空带括号列表初始化。
(2) 花括号环绕的单个初始化器初始化一个非类类型对象。（对于类类型和其他使用花括号初始化列表的初始化,见列表初始化）。(花括号唯一)
(3) 函数式类型转换 或 以带括号的表达式列表初始化纯右值临时量。
(4) static_cast 临时量。
(5) 非空初始化器 new 表达式 初始化动态存储期对象。
(6) 构造函数 初始化器列表 初始化基类 和 非静态成员。
(7) lamda 按复制捕获的变量初始化闭包对象的成员。

  • 效果
  • T 是数组(c++20前(非良构)/c++20起(聚合初始化数组，值初始化不带初始化器的元素，可窄化转换))
  • T是类类型
    • 1.同类型纯右值：(c++17起)初始化器是 纯右值表达式，且类型相同为T，则用初始化器表达式自身，而非从它实质化的临时量(即不在内存实例化临时量，不要求、不调用相应构造函数)，初始化目标对象。(C++17 前，编译器可以在此情况下消除源自纯右值的构造，但适合的构造函数(复制/移动)仍必须可访问:见复制消除)
      1. 构造函数重载决议，调用构造函数初始化对象。
    • 否则如果，T是聚合类，按 聚合初始化，但是允许窄化转换，不允许使用指派初始化器，不延长临时量的生存期，不进行花括号消除，并值初始化任何无初始化器的元素。(c++20起)
  • 否则，T是非类，但是源类型是类类型(可转换)，由源类型及其基类的转换函数，将初始化器表达式转换为所初始化的对象。
  • 否则，T是bool源类型是std::nullptr_t,则被初始化对象的值为false。

  • 否则，必要时标准转换，转换 其他对象 为T，所初始化对象的值为该值。

  • 注解 直接初始比复制初始化更宽松：复制初始化仅考虑非显式的构造函数和非显式的用户定义转换函数，而直接初始化考虑所有构造函数和所有用户定义转换函数。
    在使用直接初始化语法 (1)（带圆括号）的变量声明和函数声明之间有歧义的情况下，编译器始终选择函数声明。此消歧义规则有时是反直觉的，并且已被称为最烦人的分析。

1.2 复制初始化(Default initialization)

从另一个对象初始化对象。 * 语法
1） T obj = other_obj;
2） T obj = {other_obj};
3） f(other_obj);
4） return other_obj;
5） throw obj; catch(T obj);
6） T arr[N] = { 其他序列 };

  • 解释
  • 1.T具名变量，跟以 等号后随一个表达式所构成的初始化器时。
  • 2.T具名变量，跟以 花括号环绕的表达式所构成的初始化器时(c++11前，c++11后归为列表初始化)。
  • 3.按值传递 实参 到函数时。
  • 4.按值返回的函数 返回时。
  • 5.按值 抛出 或 捕获 异常时。
  • 6.作为 聚合初始化 的一部分，初始化每一个提供了初始化器的元素。
  • 复制初始化的效果：
  • T是类类型：(c++17起) 初始化器是纯右值表达式且类型为T(同类型)，以表达式初始化目标对象。
  • T是类类型：且其他对象(other_obj)是T或T的派生类， 调用T的非显式构造函数（即转换构造函数）。
  • T是类类型：且其他对象(other_obj)不是T或T的派生类；或T不是而其他对象是类类型；前者使用转换构造函数(纯右值直接初始化对象,可能不调用，但要求适合的构造函数(复制/移动)是可访问的)，后者使用转换函数。

  • 否则，T 和 其他对象 都不是类类型，必要时用标准转换将 其他对象的值转换成 T 的无cv限定版本。

  • 转换构造函数 不以说明符 explicit 声明且可用单个参数调用 (C++11 前)的构造函数被称为转换构造函数（converting constructor） ; 隐式声明的及用户定义的非显式(不以说明符 explicit 声明)复制构造函数与移动构造函数也是转换构造函数。
    通常说法是转换构造函数指定了一个从它的各实参类型（如果存在）到它的类类型的隐式转换。注意非显式的用户定义转换函数也指定了一个隐式转换。

  • 用户定义转换函数 用户定义的转换函数在隐式转换的第二阶段被调用，第二阶段由“零个或一个转换构造函数”或“零个或一个用户定义转换函数”构成。
    如果转换函数和转换构造函数都能用于进行某个用户定义转换(前提是同时存在)，那么复制初始化和引用初始化语境中的重载决议会同时考虑转换函数和转换构造函数，但在直接初始化语境中只考虑转换构造函数(不是只考虑，而是优先级考虑，因为本文 ###直接初始化 注解 说会考虑)。

  • 注解
    复制初始化不如直接初始化宽容：explicit 构造函数不是转换构造函数，也不会被复制初始化考虑。
    ``` struct Exp { explicit Exp(const char) {} }; // 不能从 const char 转换 Exp e1("abc"); // OK Exp e2 = "abc"; // 错误，复制初始化不考虑 explicit 构造函数

struct Imp { Imp(const char) {} }; // 可以从 const char 转换 Imp i1("abc"); // OK Imp i2 = "abc"; // OK 另外，复制初始化中的隐式转换必须从初始化器直接生成 T，然而直接初始化则存在比如期待一个从初始化器到 T 的某个构造函数实参的隐式转换的情况。 struct S { S(std::string) {} }; // 可以从 std::string 转换 S s("abc"); // OK：从 const char[4] 转换到 std::string S s = "abc"; // 错误：没有从 const char[4] 到 S 的转换 S s = "abc"s; // OK：从 std::string 转换到 S ``` 如果 其他对象 是右值表达式，那么重载决议会选择移动构造函数并在复制初始化期间调用它。这仍然被当做复制初始化；没有此种特殊情况的专门术语（如“移动初始化”）。
隐式转换以复制初始化的方式定义：如果能以表达式 E 复制初始化 T 类型的对象，那么 E 就可以隐式转换到 T。
具名变量的复制初始化中的等号 = 与赋值运算符无关。重载的赋值运算符对复制初始化没有影响。

  • 列表初始化中的隐式转换序列({}作实参如何初始化形参)
    在列表初始化中，实参是 花括号初始化器列表，但它不是表达式，所以到就重载决议而言的形参类型的隐式转换序列以下列规则决定：

    • T是聚合体，桥初始化器由一个 同类型 或 它的派生类组成，转换序列是派生类到形参类型的转换序列。
    • T是字符数组的引用，且初始化器列表拥有单个元素，元素是类型适当的字符串字面量，恒等转换。
    • T是std::initializer_list，且存在从每个初始化器列表元素到 X 的非窄化隐式转换，转换序列是所需的最坏转换。如果 花括号初始化器列表 为空，那么转换序列是恒等转换。
    • 形参是“N个T的数组”，初始化器列表必须有 N 个或更少的元素，且所用的隐式转换序列是将列表（或空花括号对，如果 {} 小于 N）的每个元素转换到 T 所需的最坏隐式转换序列。
    • T是非聚合类类型X，那么选取X的构造函数C 以从实参初始化器列表初始化：
    • C 是非initializer_list构造函数，且初始化器列表拥有单个类型为X的元素(单个同类型)，隐式转换序列具有准确匹配等级。如果该初始化器列表拥有单个元素，它具有派生自X的类型，隐式转换序列具有转换等级。（注意它和聚合体的区别：聚合体在考虑聚合初始化前直接从单元素初始化器列表进行初始化，而非聚合体在考虑任何其他构造函数之前先考虑 initializer_list 构造函数）
    • 否则，隐式转换序列是以恒等转换作为第二标准转换序列的用户定义转换序列(非显示 单实参可调用构造函数 和 非显示转换函数(没有参数和返回值))。
    • T是可按照聚合初始化从初始化器列表初始化的聚合体，隐式转换序列是以恒等转换作为第二标准转换序列的用户定义转换序列。
    • T是引用，应用引用初始化规则。
    • T不是类，且初始化器列表拥有一个元素，那么隐式转换序列是将该元素转换到形参类型所要求者。
    • T不是类，且初始化器列表没有元素，那么隐式转换序列是恒等转换。

  • 列表初始化的隐式转换规则概览：

  • 1.聚合体初始化：
    如果形参是聚合体（如结构体或联合体），且列表恰好有一个元素是该类型或其派生类，使用到该类型的转换。
  • 2.字符数组引用：
    形参是字符数组引用，且列表仅含一个字符串字面量，使用恒等转换。
  • 3.std::initializer_list 初始化：
    形参类型为 std::initializer_list，列表中的每个元素都能非窄化隐式转换为X，使用这些转换。 空列表初始化时，视为恒等转换。
  • 4.固定大小数组：
    形参为固定大小数组的引用，列表元素数量不超过数组大小，使用元素到数组类型的转换。
  • 5.未知边界数组（C++20起）：
    形参为未知边界数组的引用，转换列表每个元素到指定类型。
  • 6.非聚合类类型构造函数：
    非基于 std::initializer_list 的构造函数，优先考虑单元素列表，特别是当元素类型与类类型相匹配或派生自类类型时。 若无上述构造函数适用，考虑其他构造函数，优先级较低。
  • 7.聚合体初始化（非 std::initializer_list 构造函数场景）：
    对于可聚合初始化的类型，使用用户定义转换，其中恒等转换作为第二标准。
  • 8.引用参数：
    直接应用引用初始化规则，可能涉及到临时对象的创建。
  • 9.非类类型参数：
    若形参不是类类型且列表只有一个元素，使用该元素到形参类型的转换。 若列表为空且形参不是类类型，使用恒等转换。
  • 10.派生列表初始化器：
    当形参为聚合类型且能按聚合初始化规则处理列表时，即使形参不是引用，也允许转换，以恒等转换为第二标准。

>>>>>>>>>>>>>>>>>>>> * 当列表初始化涉及多元素，如 T t = {1,2,3}，主要遵循以下规则： * 1.std::initializer_list 初始化:
如果 T 是 std::initializer_list 类型，且列表中的每个元素都可以通过非窄化隐式转换变为 X 类型，那么这个规则适用。例如，如果 T 是 std::initializer_list，那么 {1,2,3} 可以直接用来初始化 T。 * 2.固定大小数组:
如果 T 是一个固定大小的数组类型，比如 int[3] 或者引用到这种类型的数组，如 int(&)[3]，并且列表中的元素数量与数组大小相匹配，那么每个元素会被转换到数组的元素类型（这里是 int）进行初始化。 * 3.非聚合类类型构造函数:
如果 T 是一个非聚合类类型，并且它有接受 std::initializer_list 的构造函数以外的构造函数，那么会尝试使用这些构造函数。特别是当列表只包含一个元素，且该元素类型与 T 或其基类兼容时，会考虑直接构造函数或转换构造函数。然而，如果没有这样的构造函数，且列表有多于一个元素，那么通常需要一个接受多个参数的构造函数来匹配列表中的元素。 * 4.聚合体初始化:
如果 T 是一个聚合体（如结构体或基本的类类型，没有私有或受保护的非静态数据成员，没有用户自定义构造函数、赋值运算符或析构函数），那么列表中的元素会依次尝试初始化聚合体的成员。成员的初始化顺序遵循它们在类定义中的声明顺序。如果列表元素数量超过了成员数量，超出的元素将被忽略；如果不足，则剩余成员默认初始化。 * 5.错误情况:
如果没有合适的构造函数或初始化方式来匹配多元素的列表初始化，编译器将会报告错误。 * 综上所述：
对于 T t = {1,2,3} 这样的初始化，如果 T 不是 std::initializer_list 类型，且假设 T 不是一个聚合体，那么这将尝试查找一个接受三个参数的构造函数（或者是接受一个 std::initializer_list 的构造函数，如果列表元素类型与 T 内部处理类型兼容）。如果找不到合适的构造函数，编译将失败。

1.7 零初始化

  • 定义
    将一个对象的初始值设置为零。

  • 语法(零初始化在语言中无语法，下列是可能零初始化的例子)
    (1) static T obj; (2) T();/t{}; ; T t = {}; (3) charT arr[n] = "短序列"； 1）静态或线程局部存储期不进行常量初始化的具名变量。 2）非类类型变量，和被值初始化的无构造函数的类类型的成员的值初始化序列的一部分，包括提供初始化式的聚合体的值初始化。(类类型详细见值初始化提及的零初始化) 3）短字符串字面量初始化任何字符类型数组时，零初始化剩余部分。

  • 注解
    如非局部初始化中所述，在进行任何其他初始化前， 零初始化所有未被常量初始化的静态和线程局部(C++11 起)存储期的变量。如果非类类型非局部变量的定义没有初始化式，那么默认初始化不做任何事，不修改先前零初始化的结果。

零初始化的指针是其类型的空指针值，即使空指针的值并非整型的零。

    target-expr : 被赋值的表达式
    op : +=|-=|=|/=|...
    new-value : 赋值表达式初始化子句
    1.简单赋值表达式
    2.复合赋值表达式

  • new-value是表达式：隐式转换 到 taget-expr 具有的类型。
  • new-value不是表达式：
    • target-expr 是标量类型且new-value 为空或者只有一个元素。：设T t = new-value,则 x = new-value 等价于 x = t(相当于显式类型转换再赋值)。
    • target-expr 是类类型：new-value 会作为实参传(参数传递存在隐式转换)递给 重载决议 选择的 赋值运算符函数,operator=({args...}),用{args...}列表初始化初始函数参数见隐式转换。 ```

2.2 内建的复合赋值

  • 内建复合赋值运算符表达式 target-expr op new-value 新值 的行为和表达式 target-expr = target-expr op new-value 完全一致，除了目标表达式只会求值一次。
  • 内建的简单赋值运算符对目标表达式和新值的要求依然使用。
  • 另外：
  • 对于 += 和 -=，目标表达式 的类型必须是算术类型或指向（可有 cv 限定的）完整定义的对象类型的指针。
  • 对于所有其他复合赋值运算符，目标表达式的类型必须是算术类型。
